# Temporary Directory (`tmp/`)

This directory is required for both development and production environments. It stores temporary files generated during application runtime, testing, or AI agent execution.

## Purpose
- Store temporary scripts (`tmp/agent`).
- Save debugging files or intermediate results that should not be part of the repository.
- Store runtime temporary files, such as **Neutral cache** and other application-generated transient data.
- Provide an ephemeral workspace for automated tools.

## Maintenance
It is safe to empty this directory at any time. Any required runtime files, such as template caches, will be automatically regenerated by the application as needed.

## Version Control
To keep the repository clean, the contents of this directory must be excluded from version control, except for this `README.md` file.

> [!IMPORTANT]
> Ensure your `.gitignore` file includes the following rules:


```gitignore
/tmp/*
!/tmp/README.md
```
